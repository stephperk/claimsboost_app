Location Text Format Design Options

Exploring different ways to display law firm location and distance information on search result cards

Problem Statement

The current location text on law firm cards displays as:

Current Format: "Charlotte, NC | 3.4 miles away"

Design Context

Where it appears: In the "LOCATION" section of law firm cards on the search results page

Current implementation: Plain text with a pipe separator (|) between city/state and distance

User need: Quickly understand where the law firm is located and how far it is from their search location

Design System Notes:

• Primary blue: #2563EB (used for CTAs and highlights)

• Light blue: #60A5FA (used for accents)

• Success green: #10b981 (used for verification badges)

• Gray text: #666, #999, #6b7280 (used for secondary information)

• Card style: Clean white background, rounded corners, subtle shadows

1Visual Hierarchy with Size Variation

Uses font size and weight to create a clear visual hierarchy. The city/state is displayed prominently in a larger, bolder font, while the distance is shown in a smaller, lighter weight to indicate it's secondary information.

✓ Pros

  • Clear visual hierarchy distinguishes primary from secondary info
  • Minimal design change from current implementation
  • Easy to scan at a glance
  • Works well on mobile with natural text wrapping

× Cons

  • Distance might feel de-emphasized for users who prioritize proximity
  • Less visual interest compared to other options
  • Relies heavily on typography for differentiation

⭐ Design Recommendation

Recommended Option: Option 1 - Visual Hierarchy with Size Variation

After analyzing all six options, Option 1 (Visual Hierarchy with Size Variation) is the recommended choice for the following reasons:

  • Optimal Information Hierarchy: The larger, bolder city/state naturally draws the eye first, which aligns with user behavior - most users want to know WHERE the firm is located before considering HOW FAR it is.
  • Clean & Scannable: Users can quickly scan down a list of cards and immediately identify the locations. The secondary distance information is still visible but doesn't compete for attention.
  • Mobile-Friendly: The flexible inline layout wraps gracefully on narrow screens without breaking the visual hierarchy or requiring additional layout logic.
  • Low Implementation Risk: This is a simple typography adjustment that won't conflict with other UI elements (badges, pills, icons) or require new assets.
  • Accessibility Win: Screen readers will naturally announce the city/state first, then distance, which is the logical reading order. No additional ARIA labels needed.

Why Not the Other Options?

Option 2 (Badge Style): While visually appealing, the badge may over-emphasize distance and compete with the verified badge and practice area pills.

Option 3 (Icon-Separated): The additional icon is redundant with the section location icon and adds visual complexity.

Option 4 (Stacked): Uses too much vertical space for relatively simple information, which could make cards unnecessarily tall.

Option 5 (Color-Coded): Green distance could be misleading when firms are actually far away, and conflicts with the existing verified badge color.

Option 6 (Minimal): While safe, it doesn't improve the user experience enough to justify the effort of making changes.

Implementation Notes

To implement Option 1, update the location section content styling:

  • City/State: font-size: 15px, font-weight: 600, color: #1a1a1a
  • Distance: font-size: 13px, font-weight: 400, color: #6b7280
  • Gap between elements: 8px
  • Remove pipe separator entirely for cleaner appearance
