Settlement Category Label Design Concepts

6 approaches to display practice area classifications on settlement cards

Design Challenge

Settlement cards currently use a 4px colored left border to indicate practice area categories, but users cannot identify the category without seeing the color legend. The challenge is to add a visible text label that clearly communicates the category while maintaining the card's clean, professional aesthetic and visual hierarchy.

Current Card Structure

  • 4px colored left border (category indicator)
  • Settlement type title (e.g., "Rear-end car accident")
  • Large settlement amount with gradient blue styling
  • AI OVERVIEW section with case description
  • Footer with law firm attribution and "View firm" link
  • White background, rounded corners (12px), 20px padding
  • Shadow on hover for interactive feedback

Concept 1: Corner Badge

Places a compact, rounded badge in the top-right corner of the card. The badge uses a light gradient background that matches the border color's theme, with dark text for accessibility. This approach keeps the category visible without interfering with the main content while leveraging the existing colored border system.

Pros

  • Immediately visible without cluttering main content area
  • Badge position creates clear visual anchor for scanning
  • Complementary gradient reinforces border color coding
  • Clean separation from settlement title and amount

Cons

  • Settlement title needs padding to avoid text overlap
  • Badge may feel disconnected from colored border
  • Less prominent than top-placement options

Concept 2: Top Pill with Icon

Features a rounded pill-style badge at the top of the card with an optional icon. The pill uses the full category color as background with white text, creating strong visual association with the border. This is the boldest and most visually prominent approach.

Pros

  • Strongest visual connection to colored border
  • Most prominent category labeling approach
  • Icon option adds visual interest and recognition
  • Works naturally at top of card hierarchy

Cons

  • Bold styling may compete with settlement amount
  • Adds vertical height to card
  • Requires icon library for each category

Concept 3: Inline Label with Accent Border

Displays the category as an uppercase inline label with a bottom border accent that matches the card's left border color. This creates a subtle connection between the label and the existing color coding system without using background fills or badges.

Pros

  • Most conservative and professional appearance
  • Bottom border creates elegant visual link to side border
  • Minimal space usage and visual weight
  • Easy to read and accessible

Cons

  • Less visually distinctive for quick scanning
  • Category label may get overlooked
  • Underline treatment is subtle compared to other options

Concept 4: Diagonal Corner Ribbon

Features a bold diagonal ribbon in the top-right corner displaying the category name. This creative approach adds visual interest and makes the category highly noticeable while maintaining a distinctive, premium feel. Most unconventional option.

Pros

  • Highly distinctive and memorable design element

  • Draws immediate attention to category

Concept 5: Header Tag with Icon and Border

Integrates a bordered tag with optional icon at the top of the card, positioned as a header element. Uses transparent background with colored border and text that matches the card's left border. Balances visibility with subtlety.

Pros

  • Clear visual hierarchy with tag as header element
  • Border treatment echoes card's left border nicely
  • Icon adds recognition without overwhelming
  • Professional appearance suitable for legal context

Cons

  • Takes up horizontal space at top of card
  • Border may create visual noise with card border
  • Settlement amount may appear lower in hierarchy

Design Recommendation

Primary Recommendation: Concept 5 (Header Tag with Icon and Border)

After analyzing all six concepts, Concept 5 provides the optimal balance of visibility, professionalism, and design coherence for the following reasons:

  • Clear visual hierarchy: Tag placement as a header element establishes category context before the settlement details, supporting natural top-to-bottom scanning
  • Professional aesthetic: Bordered tag with transparent background maintains the clean, trustworthy appearance required for legal settlements
  • Strong border connection: The tag's colored border directly echoes the card's left border, creating cohesive visual language that reinforces the color-coding system
  • Scalable iconography: Optional icon support allows for future enhancement with category-specific symbols to aid quick recognition
  • Accessibility friendly: Text remains on white background ensuring WCAG AA contrast compliance while the colored border provides sufficient visual distinction
  • Mobile responsive: Tag can shrink gracefully or wrap on smaller screens without breaking layout

Alternative for Consideration: Concept 3 (Inline Label with Accent Border)

If a more conservative, minimal approach is preferred, Concept 3 offers excellent readability and professional appearance while adding almost no visual weight to the card. The underline accent creates an elegant connection to the border color without badges or fills.

Not Recommended: Concept 4 (Diagonal Corner Ribbon)

While visually striking, the diagonal ribbon may appear too decorative for the serious nature of legal settlements and could undermine user trust. The angled text also presents accessibility challenges.

Implementation Considerations:

  • Ensure category names truncate gracefully with ellipsis at ~18 characters
  • Maintain 1.5px-2px border width for tag outline to match card border weight
  • Consider 8-10px of margin-bottom for tag to create breathing room before settlement title
  • Test icon visibility at 12px-14px size to ensure clarity without overwhelming text
  • On mobile (<480px), consider removing icon to save horizontal space
  • Add hover state that slightly brightens border and text color for interactive feedback
  • Consider adding the category as a data attribute for filtering and analytics
